  • storageFile Management
    • Root Folder
    • "Starting" folder on all storage mediums
    • E.g. (C:\) on Windows machines
    • "/" on Mac and Linux machines
    • Seperate directories can be created here
    • Usually managed by the OS
    •  
    • Filenames
    • Have two parts to them - filename and extension
    • Can be up to 255 characters in length
    • Extension is determined by the creation software
    • E.g. - .docx is a Microsoft Word document and .html is a web page
    • Filenames disallow special characters like /
    •  
    • File Security
    • Files can be encrypted or password protected
    • The entire hard drive can be encrypted, which requires a password to decrypt it on startup
    •  
    • Directories
    • Also called folders
    • Allow you to store other files and directories in them
    • The file path (showed in the address bar) is determined by which folders are opened
    • The folder path for program files on Windows is C:\Program Files
    • On Mac, the folder for a users applications would be /Users/user/Applications

  • appsSoftware
    • Systems Software
    • Type of software that runs computer hardware and programs
    • Provide basic functionality so that users don't have to write their own programs
    • Designed to maintain/operate the OS
    •  
    • Operating Systems
    • What is shown on a screen
    • Manages system and hardware resources
    • All services require an OS to function
    • Operating Systems include Microsoft Windows, Mac OS, Linux
    • There are lots of different variants of Linux, some of these include - Ubuntu, Linux Mint
    • Single-user operating systems allow for only one general user
    • Often found in household appliances
    • They only allow one user to customise it
    • Network Operating Systems allow multiple users to login by sending the login request to a server
    • Seperate accounts so users cannot access each other's files
    • Can be used to access shared network drives, printers etc
    • They will most often have an ADFS server that will act as a SSO service in organisations
    • This allows for Single Sign On, once a user is signed in once, they will not need to sign into other services
    • E.g. Office 365, Box, Slack
    • ADFS stands for Active Directory Federation Service

    • Application Software
    • Applications are programs that complete different tasks
    • Examples of this are - Microsoft Word, Photoshop, Skype, Atom, Sequel Pro
    • Google Chrome is an Internet Browser
    • Microsoft Word is a word processor
    • Programs are instructions that are executed by the processor
    • These applications are written in a programming language such as C# or Swift
    •  
    • Utility/System Tools Software
    • Examples of these are antiviruses, disk tools, backup software
    • They can also be system accessories - such as volume control
    • Software designed to analyse, configure and optimise a computer
    •  
    • Software Licensing
    • Licenses for software is a way of the developer checking that the user has a genuine copy of the software
    • It prevents software being re-sold as it requires a license to be activated

    • Data and Information
    • Storing data in a logical way
    • Stored digitally
    • Tables have to be created to store data
    • Planning has to be done to acknowledge where data goes
    • Fields are 'categories' that are set up
    • Each table has to have a primary key
    • Foreign key - primary key from another table

    • Data Validation
    • Presence - making sure a field hasn't been left blank
    • Type - tests if the data entered matches the field's data type
    • Length - tests if the correct number of characters have been entered
    • Range - tests if the data is between two specific values

    • Data Verification
    • Checking that the data that is inputted is as accurate as it can be to the original data collection
    • Visual Verification
    • Double entry verification
    • Typed into a computer by a user
    • Typed into a computer by a second user
    • Computer then checks if they match

    • Data Input and Output
    • Data can be inputted from a GUI - e.g. Microsoft Access or Sequel Pro
    • Data can also be inputted from web interfaces - such as phpMyAdmin
    • Web forms can be created and hosted on servers, these can send their results to a database
    • Data can be outputted to reports
    • It looks more like a document
    • It is formatted and can have styling options changed
    • Reports can be customised with company names, logos etc to make it look more personal

  • vpn_lockSecuring Data
    • Backup
    • Data can be backed up to external hard drives, servers and cloud backup solutions
    • This is done to prevent any data from being lost if there are any problems with the machine
    • It can also be used to revert to earlier versions of data if anything goes wrong with current copies
    •  
    • User Access
    • IT Administrators in a workplace environment control which users can access what files
    • In most situations, they require the user to be authenticated and only allow that user access to their files
    • File permissions can be set in shared areas to determine which users are allowed to access or edit files
    •  
    • External Threat
    • Attackers can attempt to gain control over internal computer situations
    • For example, using Brute-Force to repeatedly guess a user's password
    • If an attacker was able to get into the internal systems, they could manipulate, delete, copy files or install ransomware to force users to pay for their files
